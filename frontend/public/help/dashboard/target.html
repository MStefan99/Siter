Hosting target determines what an app should serve when a request has been received.

Target can be one of two things:

  • Local directory
  • Web server (either local or remote)

To host static files from a given directory, just choose directory as the target and pick a directory to be served. If you are using an SPA (single-page application), you can enable the SPA mode. This will adjust the routing to serve your application and avoid 404 errors.

If you have an existing web server that you want to use, choose web server. In this case an app will act as a proxy, forwarding the requests to the server and serving its responses back to the users. The server can be remote or local but using an app as reverse proxy is generally recommended.

If your backend is available on a domain different from your frontend, you need to use a CORS section. Here you will find a list of domains allowed to access your backend. If a request is coming from a matching domain, an app will add a CORS header even if it isn't supported by the target server.
