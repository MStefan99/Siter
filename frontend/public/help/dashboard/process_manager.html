Process manager allows you to skip the trouble of setting up an external process manager such as PM2 and forever and start your website server (or any other executable file on your computer) without leaving Siter.

Adding a process is pretty self-explanatory: you need to specify an application to run (for example, node or python, can also be empty), a file to use (like a location of your Node.js entry file or your executable), flags/CLI parameters and environment variables to use.

There are some environment variables that are sometimes added to every process and cannot be edited. These include:

  • PATH - Similar to PATH variable on your computer.
  • PORT - specifies which port your server should use. Port number is taken from the Target section when target is a web server. Needs server support.
  • CRASH_COURSE_URL - URL of a Crash Course instance for the server to use. The address is taken from analytics section. Needs server support.
  • CRASH_COURSE_AUDIENCE_KEY - Audience key to be used with Crash Course instance. The key is taken from analytics section. Needs server support.
  • CRASH_COURSE_TELEMETRY_KEY - Telemetry key to be used with Crash Course instance. The key is taken from analytics section. Needs server support.
