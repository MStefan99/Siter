Route target determines what Siter should serve when a request for a given route has been received.

Siter target can be one of two things:

  • A directory
  • Another web server

To host static files from a given directory, just choose directory as the target and paste the directory path to be served. Siter will automatically find the correct media type based on the file extension and send it along with the file.

If you have an existing web server that you want to use with Siter, choose web server. In this case Siter will act as a proxy, forwarding the requests to the server and serving its responses back to the users. The server can be remote or local but using Siter as reverse proxy is generally recommended.
